# AI 面试助手 -- 用户使用文档

---

## 1. 环境要求

### 1.1 Mac 端（主力设备）

| 项目 | 要求 |
|------|------|
| 系统 | macOS 12 Monterey 及以上 |
| 浏览器 | Chrome 90+ / Edge 90+ / Safari 15.4+（推荐 Chrome） |
| 网络 | 需与后端服务在同一局域网（本地部署则无需额外网络） |

**浏览器权限（首次使用时需授权）：**

| 权限 | 用途 | 何时弹出 |
|------|------|----------|
| 麦克风 | 采集你的声音（候选人音频通道） | 点击「录音」按钮时 |
| 屏幕共享 | 采集面试软件的系统音频（面试官音频通道） | 点击「录音」后自动弹出 |

> **重要**：屏幕共享弹窗中，选择面试软件窗口后，务必**勾选「共享音频」**（Share audio），否则无法采集面试官声音。

**macOS 系统权限（系统设置 -> 隐私与安全性）：**

| 权限 | 路径 | 说明 |
|------|------|------|
| 麦克风 | 隐私与安全性 -> 麦克风 | 允许浏览器访问麦克风 |
| 屏幕录制 | 隐私与安全性 -> 屏幕录制 | 允许浏览器进行屏幕共享（采集系统音频） |

如果浏览器弹窗被拒绝或未出现，请到系统设置中手动开启以上权限后**重启浏览器**。

---

### 1.2 iPhone 端

| 项目 | 要求 |
|------|------|
| 系统 | iOS 15 及以上 |
| 浏览器 | Safari（推荐） / Chrome for iOS |
| 网络 | 需与运行后端服务的电脑在**同一 WiFi** |

**iPhone 浏览器权限：**

| 权限 | 用途 | 说明 |
|------|------|------|
| 麦克风 | 采集语音 | 点击录音时 Safari 会弹出授权请求，选择「允许」 |

**iPhone 限制说明：**

| 功能 | iPhone 支持情况 | 说明 |
|------|----------------|------|
| 麦克风录音 | 支持 | Safari/Chrome 均可 |
| 系统音频采集 | 不支持 | iOS 不允许网页捕获其他 App 的音频，这是系统限制 |
| 手动输入问题 | 支持 | 底部输入框正常使用 |
| 查看转写/答案 | 支持 | Tab 切换浏览 |
| 极速/精确双模式答案 | 支持 | 两路答案同时展示 |
| 简历上传 | 支持 | 可从「文件」App 选取 PDF/DOCX/TXT |
| 添加到主屏幕 | 支持 | Safari -> 分享 -> 添加到主屏幕，可全屏使用 |

> **建议**：iPhone 端最适合作为**答案查看器**或**手动输入问题**使用。如需完整的语音采集能力（尤其是面试官音频），请使用 Mac 端。

## 2. 访问方式

### 2.1 Mac 本机访问

浏览器打开：
```
http://localhost:3000
```

### 2.2 iPhone 访问（同一 WiFi）-- 独立使用

1. 在 Mac 上查看局域网 IP：
   - 系统设置 -> Wi-Fi -> 点击已连接网络的「详细信息」-> 查看 IP 地址
   - 或终端执行：`ipconfig getifaddr en0`
2. iPhone 浏览器打开：
   ```
   http://<Mac的IP地址>:3000
   ```
   例如：`http://192.168.1.100:3000`

3. （可选）添加到主屏幕以获得全屏体验：
   - Safari 中打开页面 -> 点击底部分享按钮 -> 选择「添加到主屏幕」

### 2.3 iPhone 作为副屏（双端互联）

**使用场景**：Mac 端负责录音和处理，iPhone 端只看答案（物理隔离，面试官看不到手机内容）。

**连接步骤**：

1. Mac 端打开 http://localhost:3000，进入面试辅助主页
2. 点击左侧栏底部「手机连接」按钮，弹出 QR 码和连接码
3. iPhone 连接方式（二选一）：
   - **扫码连接**：用 iPhone 相机扫描 QR 码，自动跳转到面试副屏页面
   - **手动输入**：iPhone 打开 `http://<Mac的IP>:3000/mobile`，输入 8 位连接码
4. 连接成功后，Mac 端检测到的面试问题和 AI 答案会实时同步到 iPhone 上

**连接要求**：
- Mac 和 iPhone 必须在同一 WiFi 网络
- 后端服务需以 `--host 0.0.0.0` 启动（允许局域网访问）

## 3. 功能使用

### 3.1 面试辅助（主页）

**Mac 端布局**：左侧控制栏 + 中间转写面板 + 右侧答案面板

**iPhone 端布局**：顶部 Tab 切换「实时转写」和「AI 答案」 + 右下角浮动控制按钮

#### 开始面试（语音模式）

1. 点击「录音」按钮（iPhone 端点击右下角浮动按钮展开控制面板）
2. **Mac**：授权麦克风 -> 选择屏幕共享窗口（勾选共享音频）
3. **iPhone**：授权麦克风即可
4. 面试官提问 -> 自动转写 -> 自动检测问题 -> AI 生成答案

#### 极速/精确双模式

系统同时启动两路答案生成：

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| 极速 | 直接调用 LLM，首 token ~1s | 简单问题、需要快速参考 |
| 精确 | 先检索知识库+题库，再调用 LLM | 复杂问题、需要准确答案 |
| 双模式 | 两路同时生成，并排展示 | 默认推荐，用户取优 |

**切换方式**：
- **Mac**：左侧栏「回答模式」区域选择
- **iPhone**：浮动控制面板 ->「回答模式」

#### 语音触发词

开启后，系统在识别到候选人说出特定短语（如"让我想想"、"这个问题"）时，自动触发快答。

**默认触发词**：让我想想、这个问题、稍等一下、嗯让我思考一下、well let me think

通过 WebSocket 命令配置：`{ "command": "set_voice_trigger", "enabled": true, "phrases": [...] }`

#### 手动输入模式

底部输入框直接输入面试问题，回车发送，AI 流式输出答案。Mac 和 iPhone 均可使用。

#### 上传简历

上传简历后，AI 回答会结合你的简历内容进行个性化：
- **Mac**：左侧栏「简历」区域拖拽或点击上传
- **iPhone**：浮动控制面板 ->「简历」-> 点击上传 -> 从「文件」App 选取

支持格式：PDF / DOCX / TXT（不超过 10MB）

### 3.2 移动端副屏

连接到 PC 端 Session 后，iPhone 自动接收：
- 实时面试问题
- 极速回答（amber 色标记）
- 精确回答（emerald 色标记）
- 底部输入框可手动补充问题

### 3.3 知识库

预设常见问题的标准答案。面试时精确模式优先匹配知识库，命中则零延迟返回。

路径：底部 Tab 栏（iPhone）/ 顶部导航栏（Mac）->「知识库」

### 3.4 题库练习

40+ 道高频面试题，按分类/岗位筛选，展开查看参考答案。

### 3.5 面试记录

查看历史面试的转写记录和 AI 复盘报告。

### 3.6 笔试辅助

粘贴题目文本，AI 解答。支持编程题。

## 4. 常见问题

### Q: 双模式下只看到极速答案，没有精确答案？
确认回答模式已设为「双模式」。精确通道需要先检索知识库和题库，首 token 通常比极速慢 2-5 秒，请耐心等待。

### Q: iPhone 上看不到面试官的转写？
iOS 不支持网页采集系统音频（这是 Apple 的系统限制）。建议用「双端互联」模式：Mac 端录音，iPhone 作为副屏查看答案。

### Q: iPhone 扫码后提示连接失败？
1. 确认手机和电脑在同一 WiFi 网络
2. 确认后端服务以 `--host 0.0.0.0` 启动（允许局域网访问）
3. 检查 Mac 防火墙是否放行了 3000 和 8000 端口
4. 尝试手动输入连接码代替扫码

### Q: 浏览器没有弹出麦克风权限请求？
- **Mac**：前往系统设置 -> 隐私与安全性 -> 麦克风，确认浏览器已被允许，然后重启浏览器
- **iPhone**：前往设置 -> Safari -> 麦克风，确认已允许

### Q: 录音后状态栏显示「系统音频未开启」？
屏幕共享时未勾选「共享音频」，或选择了不支持音频共享的窗口。请重新点击录音，在共享弹窗中勾选「Share audio」/「共享音频」。

### Q: 页面打开后显示「未连接」？
后端服务未启动。在电脑终端执行：
```bash
cd backend && source venv/bin/activate && uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
